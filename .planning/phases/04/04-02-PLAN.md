  <name>Task 1: Create SetLocale middleware</name>
  <files>
    app/Http/Middleware/SetLocale.php
  </files>
  <action>
    Create `app/Http/Middleware/SetLocale.php`:

    ```php
    <?php

    namespace App\Http\Middleware;

    use Closure;
    use Illuminate\Http\Request;
    use Illuminate\Support\Facades\URL;
    use Symfony\Component\HttpFoundation\Response;

    class SetLocale
    {
        public function handle(Request $request, Closure $next): Response
        {
            $locale = $request->segment(1);

            if (!in_array($locale, ['vi', 'en'])) {
                $locale = 'vi';
            }

            app()->setLocale($locale);
            URL::defaults(['locale' => $locale]);

            return $next($request);
        }
    }
    ```

    Validation logic per D-06: if locale segment is not in ['vi', 'en'], fall back to 'vi'. This handles root-level requests that get caught by the prefix middleware but have no valid locale.

    Use `Illuminate\Support\Facades\URL::defaults()` to set locale so that `route('products.index')` generates correct URLs like `/vi/san-pham` without needing `['locale' => 'vi']` every time.
  </action>

    </automated>
  </verify>

  <action>
    Restructure `routes/web.php`:

    1. Move ALL existing frontend routes inside:
    ```php
    Route::prefix('{locale}')
        ->where(['locale' => '[a-z]{2}'])
        ->middleware(['web', SetLocale::class])
        ->group(function () {
            // ... existing routes here
        });
    ```
    Use `App\Http\Middleware\SetLocale::class` — NOT a string alias (but also register as alias in bootstrap/app.php for later use). Import: `use App\Http\Middleware\SetLocale;`

    IMPORTANT: Keep the Route::get('/', [...]) AND all other routes inside the group. Do NOT leave routes outside.

    2. Add root redirect BEFORE the locale group (outside the group):
    ```php
    Route::get('/', function () {
        $locale = 'vi';
        if (request()->server('HTTP_ACCEPT_LANGUAGE') && preg_match('/^en\b/', request()->server('HTTP_ACCEPT_LANGUAGE'))) {
            $locale = 'en';
        }
        return redirect("/{$locale}");
    });
    ```
    This is a simple closure redirect — NOT a controller method. For route caching, closures prevent full caching but for Phase 4 this is acceptable since the group prefix also prevents full route caching anyway. Per Research finding about route caching.

    3. Add robots.txt route:
    ```php
    Route::get('/robots.txt', function () {
        return response("User-agent: *\nAllow: /\nSitemap: " . url('/sitemap.xml'))
            ->header('Content-Type', 'text/plain');
    });
    ```

    4. Admin routes (`/admin/...`) stay OUTSIDE the locale prefix. They are registered by Filament's service provider and already handled separately. Do NOT modify admin routes.

    5. Remove `Route::get('/', ...)->name('home')` from its current position (it gets moved inside the group). Keep the route name `'home'`.

    Final structure:
    - Top: imports (Route, SetLocale, all controllers)
    - Root redirect (outside group)
    - robots.txt (outside group)
    - Route::prefix('{locale}')->where(...)->middleware(...)->group(function () {
        // home, products, categories, posts, projects, contact, inquiries
      })
