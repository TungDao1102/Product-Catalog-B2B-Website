    Create a new migration file `2026_06_28_000001_make_translatable_columns.php` with:

    **Column changes (up method):**
    1. **categories:** `$table->json('name')->change(); $table->json('description')->change();`
    2. **brands:** `$table->json('name')->change(); $table->json('description')->change();`
    3. **products:** `$table->json('name')->change(); $table->json('description')->change(); $table->json('short_description')->change(); $table->json('meta_title')->change(); $table->json('meta_description')->change();`
    4. **posts:** `$table->json('title')->change(); $table->json('content')->change(); $table->json('excerpt')->change(); $table->json('meta_title')->nullable()->after('excerpt'); $table->json('meta_description')->nullable()->after('meta_title');`
    5. **projects:** `$table->json('title')->change(); $table->json('content')->change(); $table->json('description')->change(); $table->json('meta_title')->nullable()->after('description'); $table->json('meta_description')->nullable()->after('meta_title');`

    **Data migration (inside the same up()):**
    For each model (Product, Category, Brand, Post, Project), iterate all records and use `getRawOriginal()` to read the existing text value before HasTranslations trait intercepts it. Then update with JSON wrapped in `'vi'` locale:

    ```php
    use Illuminate\Support\Facades\DB;

    // For Category - wrap name and description into JSON
    $models = DB::table('categories')->get();
    foreach ($models as $model) {
        DB::table('categories')
            ->where('id', $model->id)
            ->update([
                'name' => json_encode(['vi' => $model->name]),
                'description' => json_encode(['vi' => $model->description]),
            ]);
    }
    ```

    Repeat same pattern for: brands (name, description), products (name, description, short_description, meta_title, meta_description), posts (title, content, excerpt), projects (title, content, description).

    For posts.meta_title and posts.meta_description (new columns): set to `json_encode(['vi' => ''])`.
    For projects.meta_title and projects.meta_description (new columns): set to `json_encode(['vi' => ''])`.

    **down method:** Reverse column changes — change JSON columns back to original types (string/text/longText), drop the meta columns added. Use `$table->string('meta_title')->nullable()->change();` etc.

    Migration MUST use `DB::raw` with `ALTER TABLE` OR use `doctrine/dbal` (already included in Laravel for column type changes). Since we're using Laravel 13, `doctrine/dbal` is available by default.

    Important: Run `composer require doctrine/dbal` only if not already present (check composer.json). For Laravel 13, `doctrine/dbal` is typically bundled, but verify and install if needed.
  </action>
