# Phase 2 — UI Design Contract

> Visual and interaction contract for Quản lý sản phẩm (Product Management). Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> 
> **Stack:** Laravel 13 + Filament v5 Admin + Bootstrap 5 Frontend (Blade)
> **Design basis:** `template-frontend/` HTML/CSS (B2B theme), already migrated to Blade views

---

## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | Bootstrap 5 (with custom CSS variables) | `template-frontend/css/style.css`, `b2b.css` |
| Component library | Bootstrap 5 + Filament v5 (admin) | composer.json, Filament docs |
| Icon library | Font Awesome 5 Free + Bootstrap Icons 1.4.1 | template-frontend HTML headers |
| Font | **Body:** Open Sans (400, 600) — **Display/Headings:** Playfair Display (700, 900) | Google Fonts via template-frontend |
| Animation | WOW.js + Animate.css + OWL Carousel 2 | template-frontend lib/ |

### Layer Architecture

```
Frontend (public)
  └─ Bootstrap 5 base + b2b.css (custom product/category components)
  └─ Font Awesome 5 (social, utility icons) + Bootstrap Icons (UI actions)
  └─ OWL Carousel 2 (homepage product carousel)
  └─ WOW.js + Animate.css (scroll animations)

Admin (Filament panel)
  └─ Filament v5 built-in UI (TALL stack)
  └─ No custom admin theming — use Filament defaults
```

## Spacing Scale

Bootstrap 5 spacing system (base unit `0.25rem` = **4px** at default 16px font-size). All custom B2B CSS follows this convention.

| Token | Value | Usage | Evidence |
|-------|-------|-------|----------|
| xs | 4px | Icon gaps, inline padding between icon + label | b2b.css `.filter-tag` gap |
| sm | 8px | Compact spacing, product meta borders, filter gaps | b2b.css `.filter-tags { gap: 8px }` |
| md | 16px | Default padding, card-body padding, section gaps | b2b.css `.product-card-b2b .card-body { padding: 20px }` |
| lg | 24px | Section padding, filter sidebar padding, brochure padding | b2b.css `.filter-sidebar { padding: 24px }` |
| xl | 32px | Layout gaps between major columns, section title margin-bottom | b2b.css `.section-title-b2b { margin-bottom: 32px }` |
| 2xl | 48px | Major section breaks, related products margin-top | b2b.css `.related-products { margin-top: 48px }` |
| 3xl | 64px | Page-level spacing between major content blocks | Bootstrap `py-5` classes (3rem = 48px, close enough) |

**Exceptions:**
- Bootstrap `g-5` grid gutter (3rem = 48px) — used for product detail page layout
- Bootstrap `py-5` padding blocks (3rem = 48px) — used consistently across all pages
- Product card image height: 220px (desktop), 180px (tablet), 200px (mobile) — defined in b2b.css
- Product thumb size: 80×80px (desktop), 60×60px (mobile)
- Gallery nav buttons: 40×40px circle
- Action buttons (`btn-quote`): `padding: 14px 36px` with 30px border-radius
- Filter sidebar sticky top offset: `top: 100px`

## Interaction Patterns

### Pattern 1: Cascading Category Selects (Admin — Filament ProductResource)

**Trigger:** User selects a Ngành (Level 0) in the Product form
**Response:** The Nhóm (Level 1) dropdown populates with subcategories filtered by the selected parent. The Loại (Level 2) dropdown resets to empty.
**Re-trigger:** Selecting a Nhóm filters the Loại dropdown.
**Edge case — Edit mode:** On loading the Edit form, all 3 dropdowns must pre-populate by traversing `$record->category->parent?->parent?` to derive levels.

### Pattern 2: Product Gallery Navigation (Frontend)

**Trigger:** User clicks a thumbnail
**Response:** Main image updates to match the selected thumbnail. Active thumbnail gets green border + full opacity. Previous thumbnail returns to 0.6 opacity.
**Alternative trigger:** User clicks prev/next chevron buttons on main image
**Response:** Cyclical navigation through gallery array. Prev/next buttons always visible.

### Pattern 3: Product Detail Sticky Sidebar (Frontend)

**Behavior:** On desktop (≥992px), the product info sidebar with name, SKU, meta, and action buttons remains sticky at `top: 100px` while user scrolls through description, specs, and brochure sections.
**Mobile (<992px):** Sidebar becomes static, positioned below the gallery.

### Pattern 4: Category Tree Expand/Collapse (Frontend — Sidebar)

**Behavior:** Sidebar shows category hierarchy as an expandable tree.
- Level 0 (Ngành) nodes visible by default
- Level 1 (Nhóm) shown on expand — collapsed by default (D-03)
- Level 2 (Loại) shown on expanding Nhóm — collapsed by default
- Clicking a category navigates to that category page

### Pattern 5: Search Form Submit (Frontend)

**Trigger:** User types keyword and presses Enter (or clicks search icon)
**Response:** Full page GET submission to search URL with `?search={keyword}`. Server renders filtered results. No live partial updates.

### Pattern 6: Filter/Sort Interaction (Frontend — Category & Search Pages)

**Trigger:** User selects a category radio, brand checkbox, or changes sort dropdown
**Response:** Page reloads with new query parameters
**Edge case:** Combining filters (category + search + sort) — all params appended to the URL

## Key Visual Relationships

### Product Card → Product Detail Link
```
[Product Card]  ──click "Chi tiết"──→  [Product Detail Page]
   Shows: image, brand, name, desc      Shows: full gallery, meta, specs, brochure, related products
   CTA: "Báo giá" (links to contact)         CTA: "Yêu cầu báo giá" (links to contact with product)
```

### Category Navigation Structure
```
[Homepage "Danh mục" section]
  │
  ├── [All Products page (products.index)]
  │     ├── Category filter (radio buttons → /san-pham?category=slug)
  │     ├── Search box (→ /san-pham?search=keyword)
  │     └── Brand filter (checkboxes)
  │
  └── [Category page (categories.show)]
        ├── Sub-category navigation (radio buttons)
        ├── Search within category
        ├── Product grid (paginated, 9/page)
        └── Breadcrumb: Home → Products → Category
```

### Admin → Frontend Data Flow
```
Filament Admin
  ├── Create/Edit Category → DB → Frontend sidebar tree
  ├── Create/Edit Brand → DB → Frontend brand filter + product meta
  └── Create/Edit Product → DB → Frontend: gallery, specs, brochure, featured
```
